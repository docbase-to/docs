Docbase secures all outgoing webhooks using **HMAC-SHA256** signatures. This ensures that every request received by your server originated from Docbase and that its payload has not been tampered with.

## The `X-Docbase-Signature` Header

Each webhook request includes a header named `X-Docbase-Signature`. This header contains a timestamp and a signature (v1), separated by commas:

- `t`: The Unix timestamp of the request.
- `v1`: The HMAC-SHA256 signature generated by Docbase.

## Verifying Signatures

To verify a webhook, follow these three steps:

#### 1. Extract the Timestamp and Signature

Parse the `X-Docbase-Signature` header to get the values for `t` and `v1`.

#### 2. Prepare the `signed_payload` string

Concatenate the timestamp (as a string), a period `.`, and the raw JSON request body. `signed_payload = timestamp + "." + raw_json_body`

<Warning>
  You must use the raw request body exactly as received. Do not use a parsed or prettified JSON object, as extra spaces or reordering will result in a signature mismatch.
</Warning>

#### 3. Compute the HMAC-SHA256

Generate a hash using your **Webhook Secret** as the key and the `signed_payload` as the message.

#### 4. Guard against Timing and Replay Attacks

- **Constant-Time Comparison:** Compare the computed signature with the `v1` value from the header using a constant-time comparison function. This prevents **timing attacks**.
- **Timestamp Validation:** Check that the timestamp `t` is recent (e.g., within the last 5 minutes) to protect your application against **replay attacks**.
