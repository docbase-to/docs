title: "Security"
---

Because webhooks can trigger critical actions in your application, you need to verify that the events are actually coming from Docbase and haven't been tampered with. 

We secure our webhooks using **HMAC-SHA256** signatures.

## The Header

Every webhook request sent by Docbase includes a `Docbase-Signature` HTTP header. This header contains a timestamp and the cryptographic signature.

It looks like this: `Docbase-Signature: t=1710003000,v1=a1b2c3d4e5f6g7h8...`

- `t=`: The Unix timestamp of when the webhook was sent (useful to prevent replay attacks).
- `v1=`: The actual HMAC-SHA256 signature generated using your Webhook Secret.

## Step-by-Step Verification

To verify the webhook in your server, follow these steps:

**1. Get your Webhook Secret** Go to your Docbase Developer Dashboard, navigate to your Webhook settings, and click **"Reveal Secret"**. It usually starts with `whsec_`. Keep this safe!

**2. Extract the signature and timestamp** Split the `Docbase-Signature` header to extract the `t` (timestamp) and `v1` (signature) values.

**3. Compute the expected signature** Using your Webhook Secret, compute an HMAC with the SHA256 hash function. The payload you need to hash is the **exact, raw JSON string** sent in the request body.

**4. Compare the signatures** Compare the signature you computed with the `v1` signature from the header. Use a "constant-time" string comparison function to prevent timing attacks.
